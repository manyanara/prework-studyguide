git commands explained:

  • git status: checks status of branch you're working on
  • git checkout -b branchname creates a branch and switches you to it
  • git add -a will commit all files to the working branch
  • git commit -m "description" will commit changes to the repo, -m indicates an added message
  • git clone (copy url) will clone a repo to your device (think ssh key)
  • git remote - v will check hte connection between remote repo and clone

git flow

  • check which branch you're on using 'git status'
  • add your edited files to the directory using 'git add-A'
  • commit your changes to the repo on the branch you're on with 'git commit -m "describe edits"'
  • check your branch is up to date using 'git pull origin main'
  • push your edits to the main branch with 'git push origin (branch name)'
  • go to github and open the pull request
  • make sure the base is the main branch and you are comparing the edited branch
  • add descriptions for your edits and create the pull request
  • merge the pull request and you're done!
